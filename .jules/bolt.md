## 2024-05-18 - Avoid Generalized List Comprehensions in Tic-Tac-Toe Minimax Heuristics
**Learning:** For static grid evaluations (like 3x3 Tic-Tac-Toe) in highly recursive algorithms, generating lines dynamically using list comprehensions (`[[board[r][c] for r in range(3)] for c in range(3)]`) in leaf-node heuristics results in massive object allocation overhead and slows down search trees. Profiling showed `winner` taking >50% of runtime due to list allocations in `Min Max.py`.
**Action:** Replace dynamic line generation and generalized `all()` checks with hardcoded array index comparisons. A simple O(1) manual comparison per line (`board[0][0] == board[1][1] == board[2][2]`) provides a ~3.7x speedup for the full Minimax pass.

## 2024-05-19 - Optimize Permutation-Based Search (TSP) using Incremental Cost and Early Pruning
**Learning:** For permutation-based searches like the Traveling Salesperson Problem (TSP), generating the full path sequence (`[start] + list(perm) + [start]`) and fully recalculating its cost in every loop iteration incurs a massive performance penalty for larger graphs. Moreover, without early pruning, the algorithm explores branches that are definitively suboptimal.
**Action:** Instead of building complete paths and calculating their total sum, compute the path cost incrementally. Add a conditional check `if current_cost >= min_cost: break` inside the increment loop to instantly prune poor paths (assuming non-negative edges). This strategy yielded over 2x speedup on medium-sized graphs in TSP.py. Note: Explicitly handle edge cases (like 1-node graphs/empty iterators) before the main loop to prevent `IndexError` when starting cost increments with `perm[0]`.
