## 2024-05-18 - Avoid Generalized List Comprehensions in Tic-Tac-Toe Minimax Heuristics
**Learning:** For static grid evaluations (like 3x3 Tic-Tac-Toe) in highly recursive algorithms, generating lines dynamically using list comprehensions (`[[board[r][c] for r in range(3)] for c in range(3)]`) in leaf-node heuristics results in massive object allocation overhead and slows down search trees. Profiling showed `winner` taking >50% of runtime due to list allocations in `Min Max.py`.
**Action:** Replace dynamic line generation and generalized `all()` checks with hardcoded array index comparisons. A simple O(1) manual comparison per line (`board[0][0] == board[1][1] == board[2][2]`) provides a ~3.7x speedup for the full Minimax pass.

## 2024-05-19 - Optimize Brute-Force Permutations (TSP)
**Learning:** For algorithmic search implementations (like TSP or brute-force permutations), avoiding full array constructions (e.g., `list(perm)`) and incrementally calculating metrics inside the permutation iterator dramatically reduces object allocation overhead. Combining this with early pruning (exiting loops immediately if the current path cost exceeds the best known minimum cost) yields a substantial performance improvement (e.g., ~3.2x speedup). Always remember to explicitly handle empty iterator edge cases (such as 1-node graphs) before the loop to prevent `IndexError` or faulty cost values.
**Action:** When implementing brute-force permutations, iterate directly over the permutation, calculate metrics incrementally, break early when possible, and ensure base/edge cases are explicitly handled prior to iteration.
