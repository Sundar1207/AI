## 2024-11-20 - [Optimize TSP Performance]
**Learning:** For brute-force pathfinding algorithms (like traveling salesman problem) relying on permutations, constructing full arrays inside the permutation loop (e.g., `list(perm)`) and calculating the total path cost iteratively causes massive memory and CPU overhead.
**Action:** When implementing or optimizing permutation-based exhaustive search, use incremental cost calculations. Evaluate each step of the permutation, checking if the accumulated path cost exceeds the best known minimum. If so, apply early pruning (`if current_cost >= min_cost: break`) to skip processing the remainder of the current permutation, which dramatically improves performance. Also, when optimizing, don't forget to explicitly handle the edge cases of an empty iterator or single-node graphs (like `graph[start][start]`) before looping.
