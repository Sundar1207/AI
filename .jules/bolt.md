## 2024-05-18 - Early Pruning in Search Algorithms
**Learning:** For algorithmic search implementations (like TSP or brute-force permutations), avoid constructing full representations (e.g., full path arrays) inside the main loop. Incrementally calculating costs and implementing branch-and-bound logic directly on iterators yields substantial performance improvements. Furthermore, optimizing inner loops may unintentionally break edge cases like 1-node graphs (causing zero-length iterations to skip assignments).
**Action:** Always implement early pruning by incrementally accumulating metrics inside loop evaluations (`if current_cost >= min_cost: break`) instead of waiting for full path generation. Additionally, verify zero-length edge cases explicitly via tests after making iterations eager.
