## 2024-05-24 - Pre-calculate goal positions for coordinate-based heuristics
**Learning:** In search algorithms like A* using coordinate-based heuristics (e.g., Manhattan distance for sliding puzzles) where the goal state is static, computing the distance by re-scanning the goal grid dynamically causes O(N^2) inner loops which severely degrades performance. In addition, failing to initialize the starting node in the priority queue with the actual heuristic value (`f = g + h`) leads to suboptimal initial expansion.
**Action:** Always pre-calculate static goal positions into a hash map (dictionary) during initialization, replacing O(N^2) scans with O(1) lookups during the heuristic calculation. Furthermore, ensure the priority queue is seeded with `(initial_h, start_node)` rather than `(0, start_node)` to correctly order the initial search.
